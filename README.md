# Light-Browser
The original code of Light Browser

Light Browser is an open-source web browser project focused on high performance, 
minimalist design, and user privacy. Developed as a lightweight alternative 
to resource-heavy browsers, this project prioritizes a clean user interface 
and a tracker-free experience.

## Core Principles
- Privacy First: No data collection or built-in tracking scripts.
- Performance: Optimized for low RAM and CPU usage.
- Open Source: Distributed under the GPLv3 license to ensure long-term 
  transparency and freedom for all users.
- Custom Aesthetics: Built with a dark, high-contrast theme (Black/Red/Neon) 
  designed for long-range usability and modern style.

## Technical Specifications
- Frontend: HTML5, CSS3, and Vanilla JavaScript.
- Search Integration: Uses DuckDuckGo as the default search engine for 
  private browsing.
- Architecture: Custom Tab Management system for efficient navigation 
  between web frames.

## Development Status
The development of this browser started with Phase 1, which focused on building the core structure, setting up the basic HTML/CSS layout, and designing the initial user interface. Once the foundation was solid, the project moved into Phase 2, where the Electron framework was integrated to turn the web code into a functional desktop application. This phase involved configuring the main process and ensuring smooth window management.
In Phase 3, the focus shifted to core functionality, including the integration of the search engine, tab management, and optimizing the browser for media streaming. To ensure a cleaner user experience, Phase 4 introduced an integrated ad-blocking system to filter out intrusive requests. Phase 5, the project underwent a final polish and code cleanup to remove unnecessary logs and comments. Phase 6 focused on branding and distribution, where the custom logo and application icons were designed and integrated. This culminated in packaging the entire project into a professional Windows installer (.exe) for easy deployment and public use.
## Note: 
If it looks like the project was finished very quickly on GitHub, it is because the entire development process was done locally on my computer. I spent a lot of time testing and fixing bugs before making the first public commit to ensure that the stable version works perfectly for everyone.
